# AI 漫剧（短剧）创作平台 —— 重构需求文档

> ⚠️ 本文档是视频走查的忠实记录（v1）。功能设计规则的深化与优化见 **[需求文档v2-功能设计优化.md](需求文档v2-功能设计优化.md)**，开发以 v2 为准、以本文档为需求溯源。
> 来源：2026-07-15 系统讲解视频（约 57 分钟，讲解人对既有系统 MECHA / mecha.one 的完整走查）。
> 原始转写稿见 [需求讲解视频-转写稿.txt](需求讲解视频-转写稿.txt)。
> 本文档区分三类内容：**【保留】** 现有合理功能、**【问题/Bug】** 现有缺陷、**【重构】** 明确提出的改造要求。

---

## 1. 背景与目标

现有系统是一个 AI 短剧/漫剧一站式创作平台（"从脚本到成片，一站式智能生成工作流"），分为**前台**（创作工作流）和**后台**（系统管理）。系统整体业务流程和阶段划分是合理的、经过实践验证的，但存在大量交互问题、逻辑缺陷和写死的实现。本次重构的目标：

1. 保留经过验证的**九阶段制作管线**和核心业务逻辑（标签→资产→绑定→一致性）。
2. 将核心创作交互改造为**对话式（Chat）模式**（参考 Codex / Claude / 即梦的交互）。
3. 将模型/厂商接入从**代码写死**改为**后台配置、前台动态呈现**。
4. 建立**统一资产体系**（图片、声音样本、帧、成片只存一份，各处引用）。
5. 完善**积分与安全体系**（平台对接大量付费 API，必须防刷防盗）。

---

## 2. 核心业务流程（必须保留的管线）

讲解人在 35:11 与 41:06 两次总结了各阶段的职责，这是整个系统的骨架：

```
项目 → 分集 → ┌─ 流程化制作（核心）
              └─ 自由创作（ComfyUI 画布，辅助）

流程化制作九阶段：
① 剧本    ：文字 → 分镜脚本 + 镜头设计（含视频反推转文字）
② 设计    ：按标签生成 人物/场景/道具 三类独立图片资产
③ 素材    ：把设计资产按镜头绑定（一致性的关键）
④ 配音    ：逐句 TTS，用真实配音时长反推每个镜头时长
⑤ 分镜    ：绑定关系 + 时长 → 合成每镜头的静态关键图
⑥ 视频    ：关键图 + 提示词 → I2V 生成视频片段（抽卡多选一）
⑦ 美化    ：拼接/剪辑/放大/补帧/对口型 → 初步成片
⑧ 成品    ：成片展示、分享、收集反馈 → 回流修改
⑨ 素材库/历史：共享资产库与生成记录
```

每阶段产物向下游流动，绑定关系一旦确定，下游必须严格遵守（不允许"前面设计了、后面自由发挥"）。

## 3. 前台分模块需求

### 3.1 项目管理

- 【保留】项目卡片列表、搜索、排序、展示形式切换；项目归档/共享/删除；分集的复制/删除/重命名。
- 【重构】可见性逻辑：
  - 新建项目默认**私有**（不需要选择）。
  - "对组可见"改为**按成员管理**：项目内可添加/移除平台成员，权限随成员范围走；成员被移出后，其产出**保留在平台内**。

### 3.2 制作模式（分集入口）

- 【保留】每集两种模式并存：**流程化制作**（核心）+ **自由创作**。
- 自由创作现状：简版节点画布（剧本→图片→视频→音频节点连线），支持 AI 指令，可调外部 API 或 GPU 集群。此前连接不通，已修复。
- 【问题】简版画布不好用；生成结果在画布上看不到（都进了流程化制作的素材库）。
- 【重构】自由创作建议用**标准化 ComfyUI 能力**重做，成为更复杂、更标准的编排模式；核心投入仍在流程化制作。

### 3.3 剧本阶段（本次重构的重点页面，"设计要大改"）

**现状**：左侧文件列表 → 贴入剧本 → "三步生成"（拆分剧本 / 生成分镜脚本 / 提取分镜提示词）→ 分镜脚本 → 镜头设计四栏布局。

**问题**：
1. 文档上传只支持 TXT，PDF/Word 均不支持 →【重构】此上传功能直接去掉（对话框内直接贴文本/传文件）。
2. 多剧本文件（新文件1/2/3）逻辑不合理：每个剧本各自产生一整条下游分支，切换必须回剧本页，互相看不到素材。原意是"同一剧本多次生成、择优"。→ 已改为**主剧本**逻辑（保留该逻辑）：任一版本可设为主剧本，下游全部挂主剧本；其他版本仅作草稿参考；**所有版本产生的素材合并保留**，切换主剧本不丢素材。
3. 镜头拆得太碎。视频模型单次最长 15 秒（即梦/Seedance；海螺更短约 10 秒），拆碎会导致大量首尾帧转场衔接工作。→【重构】拆分策略：同场景连续镜头**默认拆成 10–15 秒**一个镜头（连续性好、可自然过场、减少转场次数）。
4. 编辑分镜脚本后再生成，会在已有镜头后**重复累加**（镜头 11 之后继续加），且编辑不联动下游已生成内容。→ 必须修复。
5. 提示词编辑窗口太小看不清，要复制到第三方工具改完再贴回来。→【重构】标准卡片式展示，逐镜头查看、就地编辑、支持中间插入新分镜。
6. 执行过程弹窗太多。→【重构】任务结束给一个 toast 通知即可。

**重构后的目标交互（讲解人明确描述，15:04–16:51）**：
- 左侧：剧本草稿列表（各版本）。
- 中间：**对话窗**（参考 Codex/Claude 新对话）。输入框可贴剧本、提要求；多轮对话修改（例："镜头太多了，改成 5 个，每个加长"→ 分镜脚本整体联动更新）。每轮生成的分镜脚本作为对话反馈结果，不满意继续对话重生成。
- 结果满意后，点"生成镜头设计"→ 右侧展开**可折叠的资产面板**（镜头设计卡片），平时折叠只留大对话窗。
- 剧本支持**按版本多次生成**，且可对某版本继续对话微调。

**分镜脚本的数据结构（保留，是后续一切的基础）**：
- 每个镜头包含四部分：**标签**（蓝=角色、深红=场景、黄=道具）、**人物对白**、**生图 Prompt**、**视频 Prompt**。
- 标签统一命名 → 同名标签在全流程绑定同一资产（同一人物/同一场景图），保证一致性、防止场景跳跃。
- 【问题】标签识别有时不准（如"办公桌"被识别为道具但应作场景）→ 保留手动添加/更正标签的能力。
- 【保留】历史版本保存与切换；插入新分镜。

### 3.4 视频反推

- 现状：独立页面。上传视频 → 扣积分 → 调本地 GPU → 输出整体提示词（中/英）、Negative、分镜脚本、结构化 JSON。只支持复制，结果还要手动贴回剧本，不好用。
- 【重构】**取消独立入口**，作为剧本阶段对话窗里的一个 skill/工具：上传视频 → 直接生成剧本草稿 → 进入标准流程。

### 3.5 设计阶段（资产设计工作台）

- 现状：从剧本标签自动生成 人物/场景/道具 三个分类的设计条目，未生成的显示红框；点 AI 生图弹出生成对话框。
- 【问题】
  1. 档位命名"化神/进阶/筑基"**完全看不懂**，不显示真实模型名，还经常调用失败。→【重构】直接透明显示模型（参考即梦选模型的方式）；不可用/未关联的模型置灰禁用并提示。
  2. 提示词润色有问题（换风格生成结果不对）。
  3. 生成过程慢且无任何反馈。→【重构】进度条 + 占位图（参考即梦：生成中先显示模糊/变形预览，完成后替换）。
- 【保留】人物/场景/道具三分类；标签卡片；本地上传（外部生成的图可传入）；角度调整（调 GPU 转 45°/侧面 90° 等，多角度人物生成）；GPU 节点选择与排队；高清放大；**去水印**（批量处理，替代 Photoshop 手工抠）。
- 【重构】交互同样改为**对话式**，可对话完善绑定图、选模型。
- 【保留（新加的功能）】**同步已有设计**：新分集一键同步之前各集已设计的角色/场景/道具，不必重做或下载重传。

### 3.6 素材阶段

- 职责：把设计资产按镜头**绑定**。可切换绑定（例：动画灯 ↔ 实景灯）。
- 【保留·关键】**锁定绑定联动**：某镜头绑定/更换资产后，后续所有用到同标签的镜头全部自动重新绑定（一致性保证）；先生成后补绑同样全局生效。
- 【Bug】绑定显示与实际不符：镜头上显示已绑定男主/女主，实际没绑定（识别错或状态错），必须修复。
- 【重构】设计页与素材页**功能按钮统一**：AI 生成、去水印、角度调整、高清放大两边都要有且行为一致（目前不同步）。
- 【保留（新加）】"本集素材 / 全部素材"切换（原来全平台素材堆在一起）。
- 每个镜头页展示：原文脚本、画面描述词（IMAGE）、视频描述词（VIDEO）+ 角色素材卡。

### 3.7 配音阶段

- 职责：按镜头逐句生成配音（角色声音/旁白，无对白的镜头旁白占位），**用真实配音时长反推镜头时长**（文字估算不准）。
- 【保留】语速调整、试听、重新生成 → 时长联动变化；BGM/AI 音效制作轨道。
- 【Bug】
  1. 配音生成后超过设计时长，剧本时长**不会自动扩展**；
  2. 出现"TTS 任务失败"；
  3. 上方逐句列表与下方时间轴时间对不上。
- 【重构】
  - 角色声音作为**资产**管理；支持批量生成 + 单句生成。
  - **时间轴必须重做**：片段时长 = f(配音时长、语速、手动设定时间、镜头内动作时间)；时间轴上要显示**镜头画面缩略 + 对白/旁白内容**，各段自动对齐；总时长自动汇总。
  - 完成后把时长与旁白**同步到分镜阶段**。

### 3.8 分镜阶段（功能较完善，保留为主）

- 职责：每镜头按绑定素材 + 画面提示词生成**静态关键图**（可多张、可重刷）。
- 【重构·关键】**模型列表由后台配置动态驱动**：后台新增图片模型（如豆包）→ 前台自动多一个选项；后台停用 → 前台消失。不允许前端写死一堆经常调用失败的模型。不同模型的参数/比例选项动态渲染。
- 【保留】质量档（自动/高/中/低）；角色一致性优先；生成后自动锁定；失败自动重试 1 次；生成结果卡片查看。
- 【保留】图上二次操作：提示词改角度/旋转、多角度人物生成、调整镜头位置、**抠图**（主体脱离/主体背景分离）、迁移/拼合/姿势模仿（丢一张姿势参考图矫正站姿）、**分镜工具**（多宫格图生成、融合、自动分镜）。
- 参考图片机制：
  - 【已修复】素材/设计绑定的图片自动作为参考图（此前失效：生成的图与绑定资产无关）。
  - 【重构】支持本地上传参考图，且**双向同步**回素材/设计 —— 全平台资产**只存一份，各处引用**。
  - 【Bug】更换绑定（如换沙发）后重新生成，仍使用旧的默认绑定 → 必须修复。
- 【保留】底部**音画联动时间轴**：关键图 + 配音对齐预览（能看到每个画面说什么话、关键帧是什么）。

### 3.9 视频阶段（I2V）

- 职责：把绑定好的关键图 + 视频提示词一键送入视频生成（无需去即梦/海螺手动传素材写要求），每镜头一个任务。
- 【重构】
  1. 档位命名同样看不懂 → 按钮化 + 后台动态模型配置（同 3.8）。选定模型后，下方按该模型要求动态呈现（要传什么图、全能模式传什么、可生成秒数等）。
  2. 整体交互模式太差，**必须重新设计**：参考即梦对话窗模式 —— 每次生成进入资产列表，**抽卡**多次生成后选定用哪个片段拼片。
- 【保留】左右双栏（任务输入/结果）可继承；两种视图（全部镜头速览 / 单镜头）；720p 生成 + **批量放大**（1080P 直接生成太贵，用 GPU 放大方案）；重新配音/上传音频；水印开关；任务队列。
- 【保留·核心】**编辑/裁剪 + 首尾帧衔接**：
  - 长场景（如 30–60 秒打斗）超过单次生成上限 → 抽取上一段**尾帧**"抽取当前帧为图片"，作为下一段的**首帧资产**，多段衔接生成连续长镜头。
  - 【Bug】该功能原本可用，疑似被近期改动破坏，需要修复。
  - 支持裁帧：画面闪动帧或尾部多余秒数可裁掉，生成裁剪版本继续向下走。
- 【新增需求】**声音样本一致性**：多段视频中同一角色的说话声音会不一致。需要像抽帧一样**抽取角色声音样本**（男一/女一）作为资产，传给后续片段的视频生成作参考，保证全片同一角色声音统一。

### 3.10 美化阶段

- 职责：把选定的片段拼接成初步成片，并做整体/单段增强。
- 【保留】右侧增强功能都很好：单段 GPU **高清放大**（至 4K）、**智能补帧**（30→60 FPS）、**对口型**（重配音/改时长后，按音频重新生成视频保证口型对上）、旁白配音、加音轨；GPU 节点选择；合成成品。
- 【重构·关键】剪辑部分完全不是标准剪辑工具，可用性太差 → **嵌入一个标准的剪辑工具（建议基于开源方案）**：视频+音频轨道剪辑、对齐、上传素材处理、导出参数设置、预览（对标剪映的体验）。没有站内剪辑，成片质量受限或被迫导出到外部工具，达不到平台定位。

### 3.11 成品阶段（新加页面，需完善）

- 现状：新加的页面（原来生成物乱堆在素材里）；展示全片成片 + 其它视频片段，可下载。
- 【重构】
  - 成品合成后展示所用片段；不满意可回美化修改重新合成。
  - 每次合成的**历史版本可查**（用了哪些素材生成的）。
  - **共享/反馈闭环**（参考即梦/剪映共享库）：成品可分享给相关人员观看、收集修改意见 → 平台内接收反馈 → 回上游修改 → 重新生成 → 再分享。

### 3.12 素材库

- 【保留（已改）】"本集素材 / 全部素材"分离；按 图片/视频/音频/抽帧素材/收藏/项目共享 分类；文件夹。
- 【重构】
  - 列表必须支持**预览**：视频要生成**缩略图/速览图**（现在只显示文件名，点开才能看）。
  - 定位为**公共/共享素材库**：其他集、其他人（有权限时）可以调用既有素材。

### 3.13 生成历史

- 【重构】增加**筛选**；支持"只看本集历史"（现在所有集堆一起，此项之前没改）。
- 【保留】显示提示词与生成物、所用模型标签；删除/下载/批量操作。

## 4. 后台需求

> 原来是三个后台（生成管理 / 系统设置 / 集成表盘）互相嵌套，现已整合为一个入口。重构时后台整体重做。

### 4.1 用户与组织

- 【保留】账号管理（启用/禁用/重置密码）、角色（super_admin / admin / user）、用户权限、项目分组、组织管理。
- 【重构】可参考我们自己已有的权限体系设计。

### 4.2 积分体系（安全重点）

- 【保留并完善】积分规则 / 积分账户 / 积分流水。
- 【重构·关键】
  - 每一种消耗性调用（生图、生视频、TTS、反推、放大、补帧……）都要可配置**扣多少积分**。目前只有"视频反推"接了积分扣减，其余全部未接入。
  - **安全性必须做好**：平台对接大量付费 API，若账号被盗且无积分限制，会被无限调用刷爆费用。积分是资金安全的闸门。

### 4.3 内容与审计

- 【保留】素材库管理；**文件回收站**（软删除 → 可预览/下载/恢复到原位置/彻底释放 OSS 空间）。
- 【重构】前台增加**用户自己的回收站**：误删可自行恢复；只有后台彻底删除后才真正不可见。
- 【保留】生成结果统计；**审计日志**：记录每个用户调用的模型类型、耗时、成功/失败、来源 IP、浏览器等。

### 4.4 数据监控

- 【保留】任务监控（状态表可重新设计，数据都有用）；生成统计（各模型调用次数、文本生成次数、生成内容量）；成本分析。
- 【重构】**集群节点使用量统计**：每个 GPU 节点的使用量目前没统计，需要补上（这是标准功能）。

### 4.5 API 厂商配置（新做的"中间服"，重点模块）

- 背景：原来厂商绑定**写死在代码里**，每加一个厂商就要改一块代码、参数也写死。现已初步做成可配置中间层，需按以下要求完善：
  - 四大类：**文本生成 / 图像生成 / 视频生成 / 语音生成**，均可在后台任意增加厂商配置卡片。
  - 同一厂商支持**多个 Key**（公用 key / 私有 key 并存），可切换启用；未生效的折叠，只展示生效的；同一厂商归组显示。
  - **连通性检测** + **真实生成测试**（发一个极小请求、产生极低费用，验证端到端真的能跑；通过后状态变更）。现有 Sora2 卡片就有"metadata 不可验证，需真实请求确认"的警告，说明只测连通不够。
  - 每张卡片：Key/Endpoint/Model 配置、控制台跳转链接、API 文档跳转链接、调用路径（API PATHS)展示（需美化完善）。
  - 现接入的厂商供参考：DeepSeek（文本）、Gemini 图像（经 laozhang.ai 网关，含快/高质量两档）、Sora2、Veo、即梦/Seedance、海螺、火山方舟等，共 23 个配置 / 12 个厂商。
- 【重构·关键】**前后台动态绑定**（讲解人 52:26–53:41 重点强调）：
  - 现状是反的：前台有什么功能，后台才能绑定什么模型（静态）。
  - 目标：**后台新建一个模型配置 → 前台对应功能区自动多出一个按钮/卡片**（可配置按钮名、支持的时长秒数、可调参数等）；后台停用 → 前台消失。例：后台加"海螺"，前台视频生成处自动出现海螺选项及其参数面板；再加"火山"，同理动态出现。

### 4.6 集群节点（GPU Agent）

- 【保留】现有机制：后台生成注册命令（含 token），复制到本地 GPU 服务器（一个完整的 ComfyUI 平台）执行即注册；**心跳模式**上报健康状态；GPU 端无需内网穿透，只要能访问平台域名即可。
- 数据流：平台把任务数据发给 GPU 本地执行（如 720P → 1080P/4K 放大），结果回传存到平台服务器。与 API 调用同构（数据过去、结果回来）。
- 已有 GPU1（legacy）/GPU2 两节点在线。

### 4.7 工作流模板（ComfyUI）

- 【保留】56 个工作流模板：JSON 上传/编辑 + **占位符配置**（image_1/image_2/prompt/seed 等映射）；分类管理（图像生成等）；一键导入。
- 【重构】增加**工作流图形化预览**：能直接看到工作流是怎么连的、按什么方式配置的。

### 4.8 费用/余额同步看板（缺失，新需求）

- 每个对接的 API 平台的**调用消耗、剩余余额、订阅套餐用量**（如火山方舟 Agent Plan 已用 7.7万/10万 AFP）应同步回本平台集中展示。
- 余额不足 / 套餐超额要**主动提醒**，并提供跳转到对应平台充值/续订的入口。
- 理由：平台涉及大量付费调用，必须有集中的费用可观测性。

## 5. 横切性重构主题（贯穿所有模块）

| 主题 | 内容 |
|------|------|
| 对话式交互 | 剧本、设计、视频等核心创作页改为对话窗模式（参考 Codex/Claude/即梦）：左侧草稿/版本列表，中间对话窗多轮生成与修改，右侧可折叠资产面板；任务完成用 toast 通知，不弹窗轰炸 |
| 统一资产体系 | 图片、参考图、声音样本、帧、视频片段等资产**全平台只存一份**，设计/素材/分镜/视频各处引用并双向同步；上传的外部资产同样入库 |
| 标签驱动一致性 | 剧本阶段产生的角色/场景/道具标签是一致性的锚点：同名标签全流程绑定同一资产；绑定变更全局联动 |
| 动态模型配置 | 所有"选模型"的地方（设计/分镜/视频/配音）都由后台厂商配置动态驱动，禁止前端写死；档位术语（化神/筑基）全部去掉，直接透明显示模型名 |
| 时长链路 | 配音真实时长 → 镜头时长 → 分镜时间轴 → 视频生成秒数 → 剪辑轨道，全链路联动，任何一处调整下游自动对齐 |
| 首尾衔接 | 长镜头 = 多段 10–15 秒生成：尾帧→下段首帧（图像连续）+ 声音样本传递（声音连续） |
| 生成体验 | 所有生成动作要有进度反馈/占位图；失败要有明确提示与重试；抽卡（多次生成择优）是标准模式 |
| 安全 | 积分闸门覆盖全部消耗性操作；审计日志全量记录；防账号盗用刷 API |
